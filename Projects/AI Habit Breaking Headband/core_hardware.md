## 概述

头箍需要一颗功能强大到能处理传感器数据并与后端通信、但体积小到可以佩戴的微控制器。关键设计理念是：**头箍本身不进行繁重的AI计算**，它只负责数据采集和指令执行。

---

## 微控制器选择

| 选项 | ESP32 | Raspberry Pi Pico W |
|---|---|---|
| **Wi-Fi** | 内置 | 内置 |
| **蓝牙** | 内置 | 内置 |
| **计算能力** | 双核240MHz，适合TinyML | 双核133MHz，适合基础任务 |
| **功耗** | 支持深度睡眠模式 | 低功耗 |
| **生态** | Arduino + ESP-IDF，社区丰富 | MicroPython / C SDK |
| **摄像头支持** | ESP32-CAM变体直接集成 | 需外接 |

### 推荐：ESP32

- ESP32-CAM变体**直接集成摄像头模块**，减少额外布线
- Arduino生态系统庞大，TinyML库支持完善
- 支持多种睡眠模式，有利于可穿戴设备的续航优化

---

## 头箍端职责

头箍端微控制器的三个核心任务：

1. **读取传感器数据** — 从IMU、麦克风、EMG、摄像头采集原始数据
2. **通过Wi-Fi流式传输** — 将传感器数据实时发送到本地服务器
3. **等待并执行指令** — 接收服务器回传的干预指令，触发对应的电机或TENS模块

```
ESP32 工作循环：

┌──→ 读取传感器 ──→ 打包数据 ──→ Wi-Fi发送到服务器
│                                       │
└──── 执行干预指令 ←── 等待服务器回传 ←──┘
```

---

## 通信协议

- **上行数据（头箍→服务器）**：IMU坐标、音频电平、压缩后的摄像头帧
- **下行指令（服务器→头箍）**：JSON格式的干预指令
  ```json
  {"action": "shock", "intensity": 3, "audio": "stop_smoking.mp3"}
  ```
