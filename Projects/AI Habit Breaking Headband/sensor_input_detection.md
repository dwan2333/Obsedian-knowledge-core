# 传感器输入检测系统

*配套文档：[AI 戒习惯头箍 (Main)](<AI Habit Breaking Headband (Main).md>)*

---

## 概述

为了高可信度地检测不良习惯并最大程度减少误报，该系统采用了**多传感器融合技术**。系统不完全依赖单一摄像头（功耗高、电池消耗快），而是通过四种传感器的综合数据进行交叉验证。

---

## ESP32-CAM（视觉检测）

| 属性 | 详情 |
|---|---|
| **核心功能** | 通过TinyML运行轻量级目标检测 |
| **检测目标** | 香烟、啤酒瓶、垃圾食品的视觉特征 |
| **功耗** | 较高 — 仅在需要时选择性激活 |
| **技术** | TinyML（边缘端机器学习推理） |

- 摄像头模块不需要持续运行，可以在其他传感器检测到可疑行为后再激活，节省电量
- 需要预先训练的模型来区分不同物体类别

---

## 6轴IMU — 加速度计/陀螺仪（运动检测）

| 属性 | 详情 |
|---|---|
| **核心功能** | 手势识别 — 追踪"手到嘴"轨迹 |
| **检测方式** | 分析运动频率和轨迹模式 |
| **功耗** | 低 — 始终开启状态 |
| **关键区分** | 抽烟与喝水有着不同的物理节奏 |

- 这是最关键的传感器之一，因为几乎所有目标习惯都涉及**手到嘴的重复运动**
- IMU数据是AI代理系统中"运动子代理"的主要输入

---

## 声学传感器/麦克风

| 属性 | 详情 |
|---|---|
| **核心功能** | 检测特定音频触发因素 |
| **检测目标** | 打火机点火声、易拉罐开启声、咀嚼声 |
| **设计特点** | 简单、轻便 |

- 作为辅助验证手段，与视觉和运动数据配合使用
- 连续咀嚼声可以帮助区分"吃东西"与"喝水"

---

## EMG传感器（肌肉活动检测）

| 属性 | 详情 |
|---|---|
| **核心功能** | 测量咬肌激活情况 |
| **放置位置** | 下巴/太阳穴附近 |
| **检测数据** | 咀嚼或吞咽的时间和力度 |

- 能够精确检测用户**何时在咀嚼以及咀嚼力度**
- 与IMU数据组合，可以高可信度地判断用户是否在进食

---

## 传感器融合策略

```
ESP32-CAM ──→ "看到香烟？"  ──┐
6轴IMU   ──→ "手到嘴动作？" ──┼──→ 多传感器融合 ──→ 置信度评分
麦克风    ──→ "打火机声？"  ──┤
EMG      ──→ "在咀嚼？"    ──┘
```

- **单一传感器不足以触发干预** — 必须多个传感器数据一致才能确认不良习惯
- 这种设计大大降低了误报率（例如：挠鼻子不会被误认为抽烟）
