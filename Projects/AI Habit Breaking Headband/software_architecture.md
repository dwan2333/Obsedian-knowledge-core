# 软件架构

*配套文档：[AI 戒习惯头箍 (Main)](<AI Habit Breaking Headband (Main).md>)*

---

## 概述

系统采用**头箍端边缘计算 + 服务器端重度AI处理**的混合架构。头箍负责数据采集和指令执行，服务器端运行Python技术栈和AI代理系统来做出智能决策。

---

## 数据管道（Data Pipeline）

```
ESP32（头箍）                  FastAPI服务器
┌──────────────┐              ┌──────────────┐
│ IMU坐标       │──────Wi-Fi──→│ 数据摄取端点  │
│ 音频电平       │              │              │
│ 压缩摄像头帧   │              │ → 视觉子代理  │
│ EMG信号       │              │ → 运动子代理  │
└──────────────┘              │ → 管理器代理  │
                              └──────┬───────┘
                                     │
                              干预指令回传
                                     │
                              {"action":"shock",
                               "intensity":3}
```

- **数据摄取**：ESP32通过本地FastAPI服务器进行实时数据流式传输
- 传输内容包括：IMU坐标、音频电平、压缩后的摄像头帧、EMG原始信号

## AI代理系统（Agent Harness）

这是系统的"大脑"所在，采用多代理架构进行协作决策：

### 视觉子代理（Sub-Agent 1 — Vision）

| 属性 | 详情 |
|---|---|
| **输入** | 压缩摄像头帧 |
| **任务** | 分析图像，判断用户是否拿着香烟/酒瓶/垃圾食品 |
| **输出** | 置信度评分 + 检测到的物体类别 |

### 运动子代理（Sub-Agent 2 — Motion）

| 属性 | 详情 |
|---|---|
| **输入** | IMU加速度计/陀螺仪数据 |
| **任务** | 分析手部运动轨迹，判断是否存在"手到嘴"动作 |
| **输出** | 动作类型分类 + 置信度 |

### 管理器代理（Manager Agent）

| 属性 | 详情 |
|---|---|
| **输入** | 视觉子代理和运动子代理的输出 |
| **任务** | 综合评估所有子代理输出，根据预设规则做出最终决定 |
| **输出** | 向ESP32发送JSON格式的执行指令 |

> **与AI代理逻辑循环的关联**：Manager Agent的决策循环与[AI代理逻辑循环](<../../Learning AI from TOP to Bottom/AI agent/the agent logic loop.md>)中描述的`while True`循环结构一致 — 持续读取输入、决策、执行，直到不再需要工具调用。

## 微调与校准

### 误报问题

最大的技术挑战是**区分健康行为和不良习惯** — 例如：AI不能在你吃健康苹果时电击你。

**解决方案 — 校准模式**：
- 系统设有**"校准模式"**，用户可以在此模式下记录日常健康行为
- 通过建立**行为基线**，AI学习用户的正常运动模式
- 只有明显偏离基线的行为才会触发干预

### 升级协议

一旦确认违规，系统执行渐进式四步升级：

| 步骤 | 干预 | 触发条件 |
|---|---|---|
| 1 | 口头警告 | 首次检测到可疑行为 |
| 2 | 头箍轻微收紧 | 用户忽略警告，行为持续 |
| 3 | 轻度电击 | 行为仍在继续 |
| 4 | 重度电击 | 多次干预后仍未停止 |
